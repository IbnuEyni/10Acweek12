**Goal:** Turn your GitHub into a portfolio that gets attention from recruiters and collaborators in under 30 seconds.

---

## 2. Pin 3–6 Strong Projects

Choose repos that show **range and depth**. Ideal mix:

| Slot | What to Pin | Why |
|------|-------------|-----|
| 1 | Your flagship project (e.g. SignalForge) | Shows end-to-end system thinking |
| 2 | A tool/library you built | Shows you can abstract and ship reusable code |
| 3 | An API or backend service | Shows production-mindset |
| 4 | A data/ML project | Shows analytical ability |
| 5 | A contribution to open source | Shows collaboration |
| 6 | A learning project with great docs | Shows growth mindset |

**How to pin:** Profile → "Customize your pins" → select up to 6.

---

## 3. Each Repo Answers: What, How, Why

Every pinned repo README must answer three questions in the first scroll:

### The Template

```markdown
# Project Name

> One-sentence description of WHAT this does and WHO it's for.

## Why

[The problem this solves. Why it exists. What gap it fills.]

## How

[Architecture overview. Key design decisions. Tech stack with reasoning.]

## Quick Start

[3–5 commands to get it running locally]

## Key Features

- Feature 1
- Feature 2
- Feature 3
```

### What Makes a README Stand Out

- **Screenshot or demo GIF** at the top (use [asciinema](https://asciinema.org/) for CLI tools)
- **Architecture diagram** for multi-component systems
- **"Lessons Learned"** section — shows reflection, not just execution
- **Status badges** (CI, coverage, version) — signals production habits

## 4. Keep Your Activity Graph Alive

The green graph signals consistency. Strategies:

- **Daily:** Even a small commit counts — fix a typo, add a docstring, refactor one function
- **Weekly:** Push at least one meaningful feature or PR
- **Automate reminders:** Set a daily "commit something" alarm

### What Counts as Activity

✅ Commits to any repo (public or private if you enable it)
✅ Opening/closing issues
✅ Pull requests (opened, reviewed, merged)
✅ Creating repos

### Low-Effort High-Signal Habits

- Document as you learn (TIL repo)
- Refactor old projects with better patterns
- Add tests to existing code
- Write GitHub Issues as specs for future features (shows planning)
